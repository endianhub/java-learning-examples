### 难点解析

1、装饰模式基本的概念

- 不改变原有代码的基础之上，新增附加功能 

2、装饰模式和代理的区别

- 代理模式是对目标方法实现增强
- 装饰模式对装饰对象实现增强

3、装饰模式应用场景

- 多级缓存设计、mybatis中一级与二级缓存、IO流

4、装饰者模式定义

- 抽象组件：定义一个抽象接口，来规范准备附加功能的类
- 具体组件：将要被附加功能的类，实现抽象构件角色接口 
- 抽象装饰者：持有对具体构件角色的引用并定义与抽象构件角色一致的接口
- 具体装饰：实现抽象装饰者角色，负责对具体构件添加额外功能。
